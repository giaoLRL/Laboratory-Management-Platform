# 实验室管理平台开发方案

> 基于 NetBox v4.6.3 框架（Django 6.x + Python 3.12+），以插件方式扩展开发

---

## 一、项目背景与目标

基于 NetBox 现有的 DCIM（数据中心基础设施管理）框架，扩展开发一套**实验室综合管理平台**，实现以下核心功能：

| 功能模块 | 描述 |
|----------|------|
| 硬件资产信息管理 | 录入、查询、统计实验室所有硬件设备（服务器、网络设备、工控机、仪器仪表等） |
| 人员任务管理 | 创建、分配、跟踪实验任务，支持状态流转与工时统计 |
| 视频上传与管理 | 上传实验过程视频、培训视频，与设备/任务关联 |
| 任务下发与通知 | 管理员向成员下发任务，系统自动通知相关人员 |

---

```
│  │  DCIM    │  Extras  │  Core (Jobs)     │ │

│  └───────────┴──────────┴─────────────────┘ │
├─────────────────────────────────────────────┤
│         PostgreSQL + Redis + 文件存储       │
└─────────────────────────────────────────────┘
```

### 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 后端框架 | Django 6.x | NetBox 内置 |
| 数据库 | PostgreSQL 15+ | 支持 JSONField、ArrayField |
| 缓存/队列 | Redis 7.x + django-rq | 后台任务与事件处理 |
| 前端 | HTMX + Bootstrap 5 + vanilla JS | NetBox 原生方案，轻量无框架 |
| API | Django REST Framework | REST API 完整支持 |
| 文件存储 | Django Media (本地) / MinIO | 视频文件存储 |
| 搜索 | PostgreSQL 全文搜索 | NetBox 内置搜索扩展 |

### 插件架构优势

选择以 **NetBox 插件** 方式开发（而非直接修改源码），优势：
- **独立升级**：NetBox 核心升级不影响业务代码
- **标准框架**：利用 NetBox 内置的权限、日志、搜索、API、Webhook 等基础设施
- **可复用**：插件可独立打包发布
- **隔离性**：插件代码在 `netbox/lab_manager/` 目录下，职责清晰

## 三、前端开发方案

> NetBox 采用**服务端渲染 + 轻量 JS 增强**的前端架构，不与 React/Vue 等 SPA 框架耦合。

### 3.1 前端技术清单

| 层级 | 技术 | 版本/源 | 说明 |
|------|------|----------|------|
| CSS 框架 | Bootstrap 5 | `netbox-external.css`（打包内置） | 全局样式、栅格、组件 |
| 图标库 | Material Design Icons (MDI) | `mdi-*` class | 6,000+ 矢量图标 |
| JS 核心 | TypeScript → `netbox.js` | 源码在 `project-static/src/` | Bootstrap 组件初始化、HTMX 钩子 |
| 动态交互 | HTMX | `htmx.org` | 无刷新表单提交、局部更新、模态框 |
| 仪表盘 | 自研 gridstack | `dashboard.ts` | 可拖拽 Widget 仪表盘 |
| 表格 | django-tables2 | 后端渲染 | 列表页标准组件 |
| 搜索选择 | Tom Select | 打包内置 | 外键下拉搜索 |

### 3.2 NetBox 前端使用规范

#### 3.2.1 严格禁止

| 禁止 | 替代方案 |
|------|----------|
| 引入 Vue / React / Alpine.js 等框架 | 使用 HTMX + Bootstrap 5 原生组件 |
| 使用 npm / yarn / webpack | NetBox 自建构建链，插件不应引入 npm 依赖 |
| 在模板中写内联 CSS | 统一定义 CSS 类，写在 `static` 中 |
| 在模板中写大段 `<script>` | JS 优先放到 `static/` ，模板仅做初始化调用 |
| 使用 jQuery | 使用 Bootstrap 5 原生 API（无 jQuery 依赖） |
| 引入第三方 CSS 组件库 | Bootstrap 5 + MDI 已覆盖全部 UI 需求 |

#### 3.2.2 模板继承链

NetBox 提供标准模板骨架，插件模板通常只需继承末级模板：

```
base/base.html            ← 根模板 (HTML5 文档结构, CSS/JS 加载)
  └─ base/layout.html     ← 侧边栏 + 导航 + 页面布局
      └─ generic/_base.html  ← 通用页面骨架 (标题/标签页/操作按钮)
          ├─ generic/object_list.html   ← 列表页
          ├─ generic/object.html        ← 详情页
          ├─ generic/object_edit.html   ← 编辑/新建页
          ├─ generic/bulk_import.html   ← 批量导入页
          ├─ generic/bulk_edit.html     ← 批量编辑页
          └─ generic/bulk_delete.html   ← 批量删除页
```

#### 3.2.3 Block 继承体系

插件模板只需覆盖以下关键 block：

| Block | 用途 | 覆盖场景 |
|-------|------|----------|
| `title` | 页面标题 (`<title>` + `<h1>`) | 每个页面必填 |
| `controls` | 右上操作按钮区 | 添加自定义按钮 |
| `tabs` | 页面标签页 | 详情页多 Tab 布局 |
| `content` | 主体内容区 | 核心业务内容 |
| `modals` | 预加载模态框 | 弹窗操作 |
| `javascript` | 页面专属 JS | 页面级 JS 初始化 |

### 3.3 HTMX 交互模式

#### 3.3.1 使用原则

NetBox 已全局集成 HTMX，只需在 HTML 属性中声明行为：

| 场景 | 实现方式 | 示例 |

#### 3.3.3 任务状态流转（拖拽改变状态）

```javascript
// lab_manager/static/js/task_board.js
// 通过 HTMX hx-vals 发送状态变更
document.addEventListener('DOMContentLoaded', function () {
  document.querySelectorAll('.task-list').forEach(function (column) {
    column.addEventListener('dragover', function (e) { e.preventDefault(); });
    column.addEventListener('drop', function (e) {
      e.preventDefault();
      var taskId = e.dataTransfer.getData('text/plain');
      var newStatus = this.closest('[id^="task-column-"]').id.replace('task-column-', '');
      // HTMX 触发服务端状态变更
      htmx.ajax('PUT', '/plugins/lab-manager/tasks/' + taskId + '/change-status/', {
        target: '#task-column-' + newStatus,
        swap: 'innerHTML',
        values: { status: newStatus }
      });
    });
  });
});
```

### 4.4 任务下发与通知系统

基于 NetBox 内置的 **EventRule + Script + Notification** 机制实现。

#### 任务下发流程

```
管理员创建任务 → 选择执行人 → 任务状态="已分配"
    ↓
触发 EventRule (task.assigned)
    ↓
执行 Script (send_task_notification)
    ↓
├─ 站内通知 (Notification)
├─ 邮件通知 (Email)
└─ Webhook 回调 (企业微信/钉钉 → 可选)
    ↓
成员查看任务 → 开始执行 → 状态流转 → 完成
    ↓
触发 EventRule (task.completed)
    ↓
通知管理员任务已完成
```

#### 自动化规则 (EventRule)

| 事件 | 触发动作 |
|------|----------|
| `task.created` | 通知管理员有新任务 |
| `task.assigned` | 通知执行人新任务已分配 |
| `task.status_changed` | 通知相关人员状态变更 |
| `task.overdue` | 定时检查，通知逾期任务 |
| `device.warranty_expiring` | 通知负责人设备即将过保 |
| `video.uploaded` | 通知相关任务人员 |

#### 自定义 Script 示例

```python
# lab_manager/scripts/notifications.py

from extras.scripts import Script, StringVar, ObjectVar

class TaskNotificationScript(Script):
    class Meta:
        name = "发送任务通知"
        description = "当任务状态变更时发送通知"

    task_id = ObjectVar(model=Task, label="任务")

    def run(self, data, commit):
        task = data['task_id']
        # 发送站内通知
        task.assigned_to.notifications.create(
            title=f"新任务: {task.title}",
            message=f"您有一个{task.get_priority_display()}优先级任务需要处理",
        )
        # 发送邮件（通过 NetBox 内置邮件后端）
        # ...
        return f"通知已发送给 {task.assigned_to.username}"
```

## 十一、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 视频大文件存储压力 | 磁盘不足 | 前端限制文件大小；采用分片上传；对接 MinIO/S3 |
| NetBox 版本升级兼容性 | 插件失效 | 锁定 NetBox 版本；CI 自动化测试兼容性 |
| 多人并发任务冲突 | 数据不一致 | 数据库悲观锁/乐观锁；任务状态机严格控制 |
| 性能瓶颈 | 响应变慢 | 数据库索引优化；Redis 缓存热数据；CDN 加速视频 |
