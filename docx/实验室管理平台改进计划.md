# 实验室管理平台改进计划

> 大学生电赛实验室管理平台  
> 保留 NetBox 框架骨架，剥离无关模块，聚焦**硬件资源管理 + 任务分配**两大核心

---

## 一、项目定位

| 项 | 说明 |
|----|------|
| 目标用户 | 电赛实验室管理员 + 普通成员 |
| 核心场景 | 购买硬件 → 录入存档 → 管理员分配任务 → 成员登录查看任务 → 上传图片/视频完成 |
| 设计原则 | 简洁实用、移动端友好、仅保留真正需要的功能 |

---

## 二、当前项目问题

| 问题 | 说明 |
|------|------|
| 引入大量无关 NetBox 模块 | DCIM（数据中心机架/电源/布线）、IPAM（IP地址/VLAN）、VPN、Wireless、Circuits（电路）、Tenancy（租户）、Virtualization（虚拟化）——与实验室场景完全无关 |
| lab_manager 插件过度设计 | 视频作为独立模块（Video 模型/缩略图/标签/分类/时长/类型）、拖拽看板、二维码、仪表盘 Widget 等都超出实际需求 |
| 硬件模型依赖 DCIM | LabDevice 通过 OneToOne 关联 DCIM Device/DeviceType/Site/Rack 等，链路冗余 |
| 前端设计复杂 | GraphQL、看板拖拽、分片上传、视频播放器等远超需求 |

---

## 三、改进策略

### 3.1 保留的 NetBox 基础设施

NetBox 的根基是 Django，其做得好的是**基础设施层**，这些保留不删：

| 保留部分 | 路径 | 作用 |
|----------|------|------|
| 用户认证系统 | `users/` + `account/` | 登录/登出/用户管理/权限 |
| 核心模块 | `core/` | 数据文件、配置、后台 Job、插件管理 |
| 扩展模块 | `extras/` | 自定义字段、标签、事件规则、脚本系统 |
| 工具库 | `utilities/` | API 工具、通用函数 |
| 前端框架 | `static/` + 模板链 | Bootstrap 5 + HTMX + MDI 图标 |
| 插件机制 | `netbox/plugins/` | 插件注册、路由、导航注入 |
| 项目管理 | `manage.py` + settings + urls 体系 | Django 标准管理 |
| 配置系统 | `netbox/configuration.py` | 数据库/Redis/插件配置 |

### 3.2 删除的 NetBox 模块

这些模块与电赛实验室无关，需要**从路由、INSTALLED_APPS、文件系统中彻底移除**：

| 删除模块 | 路径 | 原因 |
|----------|------|------|
| DCIM | `dcim/` | 数据中心基础设施管理（机架/电源/布线/站点），与实验室无关 |
| IPAM | `ipam/` | IP 地址管理（子网/VLAN/VRF/ASN），与实验室无关 |
| Circuits | `circuits/` | 通信电路管理，与实验室无关 |
| VPN | `vpn/` | VPN 隧道管理，与实验室无关 |
| Wireless | `wireless/` | 无线网络管理，与实验室无关 |
| Virtualization | `virtualization/` | 虚拟机/集群管理，与实验室无关 |
| Tenancy | `tenancy/` | 多租户管理，与实验室无关 |
| GraphQL | `netbox/graphql/` | 不需要 API 层 |
| REST API Schema | drf-spectacular | 不需要对外 API |

### 3.3 lab_manager 插件改造

保留插件架构，但精简到仅两个模型：

| 改造项 | 当前 | 改进后 |
|--------|------|--------|
| 视频模块 | 独立 Video 模型 + 缩略图 + 标签 + 分类 + 时长 | **删除 Video 模型**，视频/图片仅作为 Task 的附件 |
| 硬件模块 | LabDevice 关联 DCIM Device（OneToOne）→ Site/Rack | **独立 Hardware 模型**，不依赖 DCIM |
| 附件上传 | 无 | **新增 TaskAttachment 模型**（支持图片 + 视频） |
| 仪表盘 Widget | 注册 dashboard widget | 删除，首页用简单统计卡片 |
| 拖拽看板 | task_board.js 拖拽 | 删除，用简单状态筛选列表 |
| 二维码 | device_qrcode | 删除 |
| 视频播放器 | 独立播放页面 | 删除，附件区域直接预览/播放 |
| GraphQL | graphql 目录 | 删除 |

## 五、数据模型设计

### 5.1 Hardware（硬件资源）

**独立模型，不依赖 DCIM**，直接继承 NetBoxModel（自带 ChangeLog、标签、自定义字段、搜索）。

```python
class Hardware(NetBoxModel):
    """硬件资源"""
    name = models.CharField(max_length=200)                      # 名称
    category = models.CharField(                                 # 类别
        max_length=30,
        choices=[('mcu','单片机/开发板'),('sensor','传感器'),
                 ('power','电源/电池'),('instrument','仪器仪表'),
                 ('tool','工具'),('wire','线材/连接器'),
                 ('module','模块/外设'),('other','其他')]
    )
    model_number = models.CharField(max_length=100, blank=True)   # 型号
    manufacturer = models.CharField(max_length=100, blank=True)   # 厂家/品牌
    quantity = models.PositiveIntegerField(default=1)             # 数量
    unit_price = models.DecimalField(max_digits=10, decimal_places=2, null=True)  # 单价
    purchase_date = models.DateField(null=True, blank=True)       # 购买日期
    purchase_link = models.URLField(blank=True)                   # 购买链接（淘宝/立创等）
    status = models.CharField(                                    # 状态
        max_length=20,
        choices=[('in_use','在用'),('idle','闲置'),
                 ('repair','维修中'),('scrapped','已报废')]
    )
    storage_location = models.CharField(max_length=100, blank=True)  # 存放位置（如"柜子A-3层"）
    custodian = models.ForeignKey(User, on_delete=models.SET_NULL, null=True)  # 保管人
    image = models.ImageField(upload_to='hardware/', blank=True)  # 实物照片
    remarks = models.TextField(blank=True)                        # 备注
```

### 5.2 Task（任务）

```python
class Task(NetBoxModel):
    """实验任务"""
    title = models.CharField(max_length=200)                     # 任务标题
    description = models.TextField()                             # 任务描述
    priority = models.CharField(                                 # 优先级
        max_length=10,
        choices=[('urgent','紧急'),('high','高'),
                 ('medium','中'),('low','低')]
    )
    status = models.CharField(                                   # 状态
        max_length=20,
        choices=[('pending','待开始'),('in_progress','进行中'),
                 ('completed','已完成')]
    )
    created_by = models.ForeignKey(User, on_delete=models.CASCADE, related_name='created_tasks')
    assigned_to = models.ForeignKey(User, on_delete=models.CASCADE, related_name='assigned_tasks')
    deadline = models.DateTimeField(null=True, blank=True)       # 截止日期
    completed_at = models.DateTimeField(null=True, blank=True)   # 完成时间
```

### 5.3 TaskAttachment（任务附件）

```python
class TaskAttachment(NetBoxModel):
    """任务附件 — 图片和视频都通过此模型上传"""
    task = models.ForeignKey(Task, on_delete=models.CASCADE, related_name='attachments')
    file = models.FileField(upload_to='task_attachments/')       # 支持图片和视频
    uploaded_by = models.ForeignKey(User, on_delete=models.CASCADE)
    remark = models.CharField(max_length=200, blank=True)        # 附件说明
```

**设计原则**：不区分图片和视频，统一用一个 FileField。前端根据文件扩展名自动判断预览方式（图片用 `<img>`，视频用 `<video>`）。

| 角色 | 显示内容 |
|------|----------|
| 管理员 | 硬件统计卡片（总数/在用/闲置）+ 任务统计卡片（总数/进行中/已完成）+ 快捷入口 |

## 八、关键设计约束

| 约束 | 说明 |
|------|------|
| 不做独立视频模块 | 视频仅作为 TaskAttachment 的 FileField 上传 |
| 不做视频标签/分类/缩略图 | 附件不需要复杂的元数据管理 |
| 不依赖 DCIM | Hardware 是独立模型，不关联 dcim.Device |
| 不做看板拖拽 | 简单的状态 Tab 筛选替代 |
| 不做 GraphQL | 删除 GraphQL 目录，纯 Django 模板渲染 |
| 不做 REST API | 不对外暴露 API，删除 drf-spectacular |
| 不做二维码 | 删除二维码生成和打印 |
| 不做邮件/Webhook | 初期不做外部通知 |
| 不引入前端框架 | 仅使用 NetBox 自带的 Bootstrap 5 + HTMX |
| 附件限制 | 单个文件最大 100MB，支持 jpg/png/gif/mp4/webm |

## 九、改进前后对比

| 维度 | 改进前 | 改进后 |
|------|--------|--------|
| 无关 NetBox 模块 | 8 个大模块（DCIM/IPAM/VPN/等） | 0 |
| 业务模型 | 5 个（LabDevice/LabDeviceType/TaskType/Task/Video） | 3 个（Hardware/Task/TaskAttachment） |
| 视频管理 | 独立模块（列表/上传/播放/标签/分类/缩略图） | 仅作为任务附件 |
| 硬件模型依赖 | LabDevice → DCIM Device → Site/Rack/Manufacturer | Hardware 独立模型 |
| 前端复杂度 | GraphQL + 看板拖拽 + 二维码 + 仪表盘 Widget | Bootstrap 列表 + 表单 + 附件预览 |
| API | REST API + GraphQL | 无（纯服务端渲染） |
