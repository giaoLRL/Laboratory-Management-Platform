# 实验室管理平台扣子联调配置说明

> 适用项目：`d:\实验室管理平台\netbox-main`
>
> 目的：说明当前 `lab_manager` 插件接入扣子所需的最小配置、接口路径与联调方式。

---

## 一、当前已具备的能力

项目当前已经完成以下基础能力：

1. `CozeGateway` 网关封装
2. 6 个 agent API 接口
3. 智能体助手页面
4. 两段式导入批次模型

当前可直接访问的页面入口：

```text
/plugins/lab-manager/agent/
```

当前可直接调用的接口前缀：

```text
/plugins/lab-manager/api/agent/
```

---

## 二、需要补充的插件配置

请在 `netbox/netbox/configuration.py` 中的 `PLUGINS_CONFIG` 里为 `lab_manager` 增加以下配置：

```python
PLUGINS_CONFIG = {
    'lab_manager': {
        # Agent API 内部鉴权
        'agent_api_token': 'replace-with-internal-service-token',

        # Coze OpenAPI 配置
        'coze_api_base_url': 'https://api.coze.cn',
        'coze_pat_token': 'replace-with-your-coze-pat',
        'coze_bot_id': 'replace-with-your-bot-id',
        'coze_timeout': 60,

        # 工作流别名映射
        'coze_workflow_ids': {
            'hardware_query': 'replace-with-workflow-id',
            'hardware_gap_analysis': 'replace-with-workflow-id',
            'task_video_search': 'replace-with-workflow-id',
            'hardware_import_validate': 'replace-with-workflow-id',
            'hardware_import_commit': 'replace-with-workflow-id',
            'assistant_chat': 'replace-with-workflow-id',
        },
    }
}
```

---

## 三、每个配置项的作用

| 配置项 | 说明 |
|---|---|
| `agent_api_token` | 扣子工作流调用平台 agent API 时使用的内部令牌 |
| `coze_api_base_url` | 扣子 API 基础地址，默认国内版 `https://api.coze.cn` |
| `coze_pat_token` | 扣子 PAT，需具备 `chat`、`getChat`、`listMessage`、`run` 等权限 |
| `coze_bot_id` | 默认用于对话模式的 Bot ID |
| `coze_timeout` | 网关请求超时时间，单位秒 |
| `coze_workflow_ids` | 工作流别名与真实 Workflow ID 的映射表 |

## 四、智能体助手页面怎么用

### 4.1 页面入口

登录系统后，进入：

```text
/plugins/lab-manager/agent/
```

也可以从左侧菜单 `实验室 -> 智能体 -> 智能体助手` 进入。

### 4.2 页面模式

页面支持两种模式：

1. Bot 对话模式
   - 工作流别名留空
   - 直接调用 `CozeGateway.run_chat_and_wait_answer()`

2. 工作流模式
   - 在输入框左侧填写工作流别名，例如 `hardware_query`
   - 页面将调用 `CozeGateway.run_workflow()`

### 4.3 建议先验证的流程

先从最简单的两条开始：

1. Bot 对话模式
   - 输入：`现在实验室有哪些 STM32 开发板？`

2. 工作流模式
   - `workflow_alias = hardware_query`
   - 输入：`查询当前空闲传感器库存`

## 五、扣子工作流调用平台 API 的请求头

当扣子工作流需要调用平台 agent API 时，建议统一附带：

```http
Content-Type: application/json
X-Agent-Source: coze-workflow
X-Agent-Token: {agent_api_token}
X-Request-ID: {request_id}
X-User-ID: {current_user_id}
```

说明：

1. `X-Agent-Token` 必须与 `PLUGINS_CONFIG['lab_manager']['agent_api_token']` 一致
2. `X-User-ID` 用于平台按当前用户权限过滤数据
3. 如果工作流运行在管理员上下文下，可以传管理员用户 ID

## 六、联调顺序建议

建议按下面顺序联调：

1. 先验证 `智能体助手页面 -> CozeGateway -> Bot`
2. 再验证 `扣子工作流 -> /plugins/lab-manager/api/agent/hardware/search/`
3. 然后验证 `扣子工作流 -> 两段式导入 validate`
4. 最后验证 `commit`

原因是：

1. 页面联调最容易看出是否是 Token、Bot ID、权限问题
2. 业务接口联调能尽快确认数据结构是否匹配
3. 导入链路涉及写操作，适合最后验

## 七、运行前注意事项

### 7.1 数据库迁移

因为已经新增了以下模型，所以你必须执行迁移：

1. `HardwareImportBatch`
2. `AgentConversation`
3. `AgentMessage`

执行命令：

```bash
python manage.py makemigrations
python manage.py migrate
```

### 7.2 扣子 PAT 权限

至少应确认 PAT 具备以下能力：

1. `chat`
2. `getChat`
3. `listMessage`
4. `run`

如果缺少权限，页面会出现“调用成功但取不到结果”或“工作流无法执行”的情况。

### 7.3 会话返回为空的常见原因

如果智能体助手页面发消息后没有文本回答，常见原因有：

1. `coze_bot_id` 配错
2. PAT 没有 `getChat` 或 `listMessage` 权限
3. Bot 没有发布为 API 服务
4. Bot 本身依赖工具，但工具权限或配置不完整
