# 实验室管理平台扣子工作流映射模板

> 用途：给扣子空间工作流编排时直接参考的参数映射模板、工具调用模板，以及 `configuration.py` 可粘贴配置块。

---

## 一、`configuration.py` 可直接粘贴配置块

请在 `d:\实验室管理平台\netbox-main\netbox\netbox\configuration.py` 中补充：

```python
PLUGINS_CONFIG = {
    'lab_manager': {
        # 扣子工作流调用本系统 agent API 时使用
        'agent_api_token': 'replace-with-internal-agent-token',

        # 扣子 OpenAPI 基础配置
        'coze_api_base_url': 'https://api.coze.cn',
        'coze_pat_token': 'replace-with-your-coze-pat-token',
        'coze_bot_id': 'replace-with-your-bot-id',
        'coze_timeout': 60,

        # 工作流别名 -> Workflow ID
        'coze_workflow_ids': {
            'hardware_query': 'replace-with-workflow-id-1',
            'hardware_gap_analysis': 'replace-with-workflow-id-2',
            'task_video_search': 'replace-with-workflow-id-3',
            'hardware_import_validate': 'replace-with-workflow-id-4',
            'hardware_import_commit': 'replace-with-workflow-id-5',
        },
    }
}
```

如果你原来已经有 `PLUGINS_CONFIG`，只需要把 `lab_manager` 这部分合进去，不要整段覆盖其他插件配置。

## 二、工作流别名建议

当前项目中，前后端和文档已经默认使用以下别名：

| 工作流别名 | 用途 |
|---|---|
| `hardware_query` | 硬件库存查询 |
| `hardware_gap_analysis` | 产品硬件缺口分析 |
| `task_video_search` | 已完成任务视频检索 |
| `hardware_import_validate` | 导入预校验 |
| `hardware_import_commit` | 导入正式提交 |

这些别名会被：

1. 智能体助手页面复用
2. `CozeGateway.get_workflow_id()` 读取
3. 后续联调文档和工作流节点配置复用

## 三、扣子调用平台 API 的公共模板

### 3.1 请求地址模板

```text
{{base_url}}/plugins/lab-manager/api/agent/...
```

### 3.2 请求头模板

```json
{
  "Content-Type": "application/json",
  "X-Agent-Source": "coze-workflow",
  "X-Agent-Token": "{{agent_api_token}}",
  "X-Request-ID": "{{request_id}}",
  "X-User-ID": "{{user_id}}"
}
```

### 3.3 变量建议

建议在扣子工作流开始节点统一定义：

| 变量名 | 类型 | 说明 |
|---|---|---|
| `user_query` | string | 用户原始输入 |
| `user_id` | string | 当前用户 ID |
| `request_id` | string | 请求追踪 ID |
| `base_url` | string | 你的系统地址 |
| `agent_api_token` | string | 平台 agent API 内部令牌 |

## 四、工作流模板一：硬件库存查询

### 4.1 适用问题

1. 现在实验室有哪些硬件？
2. 现在有哪些空闲开发板？
3. 某类硬件库存还有多少？

### 4.2 节点建议

```text
开始
  ->
意图判断
  ->
参数抽取
  ->
HTTP 调用 search_hardware
  ->
LLM 整理结果
  ->
结束
```

### 4.3 参数抽取结果模板

```json
{
  "keywords": ["stm32", "开发板"],
  "category": "mcu",
  "status": "idle",
  "approval_status": "approved",
  "limit": 10,
  "offset": 0
}
```

### 4.4 HTTP 节点配置模板

```json
{
  "method": "POST",
  "url": "{{base_url}}/plugins/lab-manager/api/agent/hardware/search/",
  "headers": {
    "Content-Type": "application/json",
    "X-Agent-Source": "coze-workflow",
    "X-Agent-Token": "{{agent_api_token}}",
    "X-Request-ID": "{{request_id}}",
    "X-User-ID": "{{user_id}}"
  },
  "body": {
    "keywords": "{{keywords}}",
    "category": "{{category}}",
    "status": "{{status}}",
    "approval_status": "approved",
    "limit": 10,
    "offset": 0
  }
}
```

## 五、工作流模板二：产品硬件缺口分析

### 5.1 适用问题

1. 我想做一个智能小车，还缺什么？
2. 我想做一个温室监测系统，需要采购哪些硬件？

### 5.2 节点建议

```text
开始
  ->
LLM 抽取产品需求
  ->
生成 requirements 数组
  ->
HTTP 调用 analyze_hardware_gap
  ->
LLM 生成采购建议与方案说明
  ->
结束
```

### 5.3 `requirements` 生成模板

```json
{
  "project_name": "智能温室监测系统",
  "requirements": [
    {
      "name": "ESP32 开发板",
      "category": "mcu",
      "required_quantity": 2,
      "keywords": ["esp32", "wifi 开发板"]
    },
    {
      "name": "温湿度传感器",
      "category": "sensor",
      "required_quantity": 4,
      "keywords": ["温湿度", "dht", "sht"]
    }
  ]
}
```

### 5.4 HTTP 节点模板

```json
{
  "method": "POST",
  "url": "{{base_url}}/plugins/lab-manager/api/agent/hardware/gap-analysis/",
  "headers": {
    "Content-Type": "application/json",
    "X-Agent-Source": "coze-workflow",
    "X-Agent-Token": "{{agent_api_token}}",
    "X-Request-ID": "{{request_id}}",
    "X-User-ID": "{{user_id}}"
  },
  "body": "{{requirements_payload}}"
}
```

## 六、工作流模板三：任务视频检索

### 6.1 适用问题

1. 帮我找出最近一周已完成任务的视频
2. 张三完成的任务里有哪些上传了视频？

### 6.2 推荐做法

推荐拆成两段：

1. 先调 `search_tasks`
2. 再根据任务条件或任务 ID 调 `search_task_videos`

### 6.3 `search_tasks` 请求模板

```json
{
  "assigned_to": "{{assigned_to}}",
  "status": "completed",
  "keyword": "{{keyword}}",
  "date_from": "{{date_from}}",
  "date_to": "{{date_to}}",
  "limit": 20,
  "offset": 0
}
```

### 6.4 `search_task_videos` 请求模板

```json
{
  "assigned_to": "{{assigned_to}}",
  "status": "completed",
  "keyword": "{{keyword}}",
  "date_from": "{{date_from}}",
  "date_to": "{{date_to}}"
}
```

## 七、工作流模板四：两段式导入第一步 `validate`

### 7.1 适用问题

1. 请帮我把这份硬件清单预校验
2. 先检查这批数据能不能导入

### 7.2 节点建议

```text
开始
  ->
解析表格/文本
  ->
生成 items 数组
  ->
HTTP 调用 validate_hardware_import
  ->
展示 batch_id、错误、重复项、待导入项
  ->
结束
```

### 7.3 请求模板

```json
{
  "source_type": "json",
  "import_action": "validate",
  "items": "{{parsed_items}}"
}
```

### 7.4 结果解释模板

在扣子里建议这样输出：

1. `batch_id`
2. 通过条数
3. 重复条数
4. 错误条数
5. 是否建议继续导入

---

## 八、工作流模板五：两段式导入第二步 `commit`

### 8.1 触发条件

只有用户明确确认后，才允许进入此步。

例如：

1. “确认导入”
2. “继续入库”
3. “使用刚才那个 batch_id 提交”

### 8.2 请求模板

```json
{
  "import_action": "commit",
  "batch_id": "{{batch_id}}",
  "confirm": true
}
```

### 8.3 结果解释模板

建议输出：

1. 导入是否成功
2. 成功条数
3. 失败条数
4. 新创建的硬件 ID 列表

## 九、智能体助手页面与工作流别名的对应关系

当前页面 `智能体助手` 中：

1. 工作流别名留空
   - 走 Bot 对话

2. 工作流别名填写为如下值时
   - `hardware_query`
   - `hardware_gap_analysis`
   - `task_video_search`
   - `hardware_import_validate`
   - `hardware_import_commit`

页面会走：

```python
CozeGateway.run_workflow(workflow_alias=...)
```

如果你后面新增工作流，只需要：

1. 在 `PLUGINS_CONFIG['lab_manager']['coze_workflow_ids']` 中增加别名
2. 在页面中输入对应别名

不需要再改网关代码。

## 十、推荐上线顺序

1. 先上线 `hardware_query`
2. 再上线 `hardware_gap_analysis`
3. 再上线 `task_video_search`
4. 最后上线 `hardware_import_validate` 和 `hardware_import_commit`

原因：

1. 前三项以读操作为主，联调风险低
2. 导入涉及写操作和批次状态，适合最后上线
