# 实验室管理平台扣子接口调用示例

> 用途：给扣子工作流、Postman、前端联调直接复用的请求示例。
>
> 当前接口前缀：`/plugins/lab-manager/api/agent/`

---

## 一、通用请求头

当扣子工作流从服务端调用这些接口时，建议统一附带以下请求头：

```http
Content-Type: application/json
X-Agent-Source: coze-workflow
X-Agent-Token: replace-with-agent-api-token
X-Request-ID: req_20260620_001
X-User-ID: 1
```

说明：

1. `X-Agent-Token` 要与 `PLUGINS_CONFIG['lab_manager']['agent_api_token']` 保持一致。
2. `X-User-ID` 表示当前调用上下文的用户 ID。
3. 管理员类写操作建议传管理员用户 ID。

## 八、常见错误示例

### 8.1 鉴权失败

```json
{
  "success": false,
  "message": "网关鉴权失败",
  "error": {
    "code": "40101"
  }
}
```

### 8.2 批次重复导入

```json
{
  "success": false,
  "message": "批次已导入，禁止重复提交",
  "error": {
    "code": "40902"
  }
}
```

### 8.3 枚举值非法

```json
{
  "success": false,
  "message": "status 不合法",
  "error": {
    "code": "40003"
  }
}
```

## 九、推荐联调顺序

1. 先调 `hardware/search`
2. 再调 `tasks/search`
3. 再调 `tasks/videos`
4. 再调 `hardware/gap-analysis`
5. 最后调 `import/validate` 和 `import/commit`
